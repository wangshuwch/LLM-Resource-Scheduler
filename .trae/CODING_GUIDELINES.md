# Python 编码规范 - 运算符使用指南

## 重要提示

**此项目中曾出现的问题：** 在代码生成过程中，Python运算符被错误地替换为HTML实体表示。

**根本原因：** 代码生成工具/IDE将 `<`、`>`、`&` 等字符自动转义为HTML实体。

**本文件旨在防止此类问题再次发生。**

---

## 问题表现

### 错误的写法（HTML实体）

```python
# 错误！这些不是有效的Python代码
if x &gt;= y:  # 应该是 >=
if x &lt;= y:  # 应该是 <=
if x &amp; y:   # 应该是 &
if x != y:   # 应该是 !=
if x == y:   # 应该是 ==
```

### 正确的写法（原生Python运算符）

```python
# 正确！使用原生Python运算符
if x >= y:
if x <= y:
if x & y:
if x != y:
if x == y:
```

|----------|--------------|------|

| `&ge;` | `>=` | 大于等于 |

## 验证和修复流程

### 1. 生成代码后检查

每次生成Python代码后，必须检查：

```bash
# 检查所有Python文件
grep -rE '&lt;|&gt;|&amp;|&le;|&ge;|&ne;|&eq;' --include="*.py" .
```

如果有输出，说明存在HTML实体转义问题，需要修复。

### 2. 手动修复

如果发现问题，手动替换：

| 搜索 | 替换为 |
|------|--------|
| `&lt;` | `<` |
| `&gt;` | `>` |
| `&amp;` | `&` |
| `&le;` | `<=` |
| `&ge;` | `>=` |
| `&ne;` | `!=` |
| `&eq;` | `==` |

### 3. 语法验证

修复后，验证代码可以正常运行：

```bash
# 编译检查
python -m py_compile your_file.py

# 或运行测试
python -m pytest tests/
```

## 最佳实践

### 代码生成时

1. **直接使用原生运算符**：不要经过任何HTML转义
2. **避免Markdown/HTML渲染**：在代码块中不要让IDE自动转义
3. **使用纯文本编辑器**：必要时使用简单的文本编辑器编写代码

### 代码审查时

1. **检查运算符**：特别注意 `<`、`>`、`&` 等字符
2. **运行测试**：确保代码可以正常编译和运行
3. **验证功能**：确保逻辑正确性

### 版本控制前

1. **最终检查**：在提交前再次检查
2. **运行完整测试**：确保所有测试通过
3. **检查差异**：使用 `git diff` 检查变更
