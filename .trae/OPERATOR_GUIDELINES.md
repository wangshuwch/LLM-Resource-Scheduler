# Python运算符编码规范

## 问题描述

在Python代码生成过程中，必须严格避免使用HTML实体表示Python运算符。所有Python运算符必须使用其原生符号表示。

## 常见错误对照表

| HTML实体 | 应该替换为的Python运算符 | 说明 |
|----------|--------------------------|------|
| `&lt;` | `<` | 小于号 |
| `&gt;` | `>` | 大于号 |
| `&amp;` | `&` | 和号 |
| `&le;` | `<=` | 小于等于 |
| `&ge;` | `>=` | 大于等于 |
| `&ne;` | `!=` | 不等于 |
| `&eq;` | `==` | 等于 |

## Python运算符列表（必须使用原生符号）

### 比较运算符
- `==` - 等于
- `!=` - 不等于
- `<` - 小于
- `>` - 大于
- `<=` - 小于等于
- `>=` - 大于等于

### 算术运算符
- `+` - 加
- `-` - 减
- `*` - 乘
- `/` - 除
- `%` - 取模
- `**` - 幂
- `//` - 整除

### 赋值运算符
- `=` - 赋值
- `+=` - 加等于
- `-=` - 减等于
- `*=` - 乘等于
- `/=` - 除等于
- `%=` - 取模等于

### 位运算符
- `&` - 按位与
- `|` - 按位或
- `^` - 按位异或
- `~` - 按位取反
- `<<` - 左移
- `>>` - 右移

### 逻辑运算符
- `and` - 逻辑与
- `or` - 逻辑或
- `not` - 逻辑非

### 成员运算符
- `in` - 在序列中
- `not in` - 不在序列中

### 身份运算符
- `is` - 是同一对象
- `is not` - 不是同一对象

## 验证步骤

在代码生成完成后，必须执行以下验证步骤：

### 1. 运算符检查
使用grep命令检查所有Python文件：

```bash
grep -rE '&lt;|&gt;|&amp;|&le;|&ge;|&ne;|&eq;' --include="*.py" .
```

### 2. 手动修复
如果发现问题，手动替换：

| 搜索 | 替换为 |
|------|--------|
| `&lt;` | `<` |
| `&gt;` | `>` |
| `&amp;` | `&` |
| `&le;` | `<=` |
| `&ge;` | `>=` |
| `&ne;` | `!=` |
| `&eq;` | `==` |

### 3. 语法验证
确保修复后的代码可以正常运行：

```bash
python -m py_compile your_file.py
```

或者运行完整的测试套件：

```bash
python -m pytest tests/
```

## 最佳实践

1. **代码生成时**：直接使用Python原生运算符，不要经过任何HTML转义
2. **字符串表示**：在字符串或注释中需要表示HTML实体时，才使用转义形式
3. **自动化检查**：每次代码生成后，都运行检查工具
4. **版本控制前**：在提交代码前，确保所有运算符检查通过

## 检查命令使用说明

### 检查单个文件
```bash
grep -E '&lt;|&gt;|&amp;|&le;|&ge;|&ne;|&eq;' your_file.py
```

### 检查整个目录
```bash
grep -rE '&lt;|&gt;|&amp;|&le;|&ge;|&ne;|&eq;' --include="*.py" .
```

## 后果

如果不遵循此规范，可能导致：
- 语法错误（SyntaxError）
- 代码无法运行
- 难以调试的问题
- 代码质量下降
